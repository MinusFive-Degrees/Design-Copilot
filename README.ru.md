# Design Copilot

Design Copilot — это расширение-рабочая панель для JLCEDA, которое объединяет проверку схемы, проверку PCB, анализ сетей, просмотр отчетов и AI-помощник в одном GUI.

## Назначение проекта

Этот проект не заменяет сам EDA-редактор. Его задача — добавить в процесс проектирования быстрый слой проверки и повторного анализа:

- быстро находить проблемы с позиционными обозначениями, футпринтами, готовностью BOM и рисками DRC на этапе схемы
- анализировать компоненты, площадки, переходные отверстия, трассы, заливки и плотные сети на этапе PCB
- сохранять отчеты для нескольких итераций проверки и сравнения
- использовать AI Agent для сводки последнего отчета, рекомендаций по исправлению, чек-листов проверки и произвольных запросов

## Текущие возможности

- Единый workbench
  В меню Home, Schematic и PCB доступен только `Open Workbench`. Все проверки, сетевые инструменты, просмотр отчетов, параметры и AI-функции собраны в одном GUI и не разбросаны по разным пунктам меню.
- Инструменты схемы
  Доступны комплексная проверка, быстрая проверка, schematic DRC и snapshot выделения. Проверка собирает статистику по компонентам, проводам, шинам, тексту, сетевым маркерам, префиксам позиционных обозначений, готовности BOM, отсутствующим designator и отсутствующим footprint, а затем формирует оценку и рекомендуемые действия.
- Инструменты PCB
  Доступны комплексная проверка, быстрая проверка, PCB DRC, snapshot выделения и анализ плотных сетей. Проверка считает компоненты, площадки, переходные отверстия, трассы, дуги, заливки, fill, region и текст, анализирует горячие сети, полноту supplier/manufacturer данных и риск по количеству via, затем формирует оценку конструкции.
- Инструменты анализа сетей
  В PCB-контексте можно автоматически подсветить самую плотную сеть, выбрать сеть из списка горячих сетей или сразу сфокусироваться по имени сети. Это удобно для проверки питания, земли и критических высокоскоростных линий.
- Система отчетов
  Каждая комплексная проверка, быстрая проверка, запуск DRC и snapshot выделения формируют отчет в едином формате. Workbench показывает последний отчет на основной сцене и хранит последние 8 записей истории для сравнения между итерациями.
- Управление параметрами
  Через GUI можно менять `Top Net Count`, `Dense Net Threshold`, `Selection Preview Limit` и `Via Risk Threshold`. Эти значения напрямую влияют на рейтинг горячих сетей, длину предпросмотра snapshot и подсказки о рисках в PCB-отчетах.
- AI-помощник
  Поддерживаются собственный endpoint, имя модели, API Key, дополнительный JSON headers, system prompt и temperature. Встроены действия `сводка последнего отчета`, `рекомендации по исправлению`, `чек-лист проверки` и `произвольный запрос`, причем вместе с запросом автоматически отправляются последний отчет, текущий контекст документа, информация о плотных сетях и активные пороги.

## Источники данных

- Статистика проекта берется из API семейств `eda.sch_*`, `eda.pcb_*` и `eda.dmt_*`
- Последние отчеты, история, настройки и AI-конфигурация сохраняются через `eda.sys_Storage`
- AI-запросы отправляются через `eda.sys_ClientUrl.request`

> Перед использованием AI-помощника нужно включить для расширения разрешение на внешнее взаимодействие в JLCEDA и убедиться, что целевой endpoint разрешает CORS. Текущий формат запроса совместим с OpenAI `chat/completions`.
