# Design Copilot

Design Copilot — это единый workbench-плагин для JLCEDA. Он объединяет проверку схемы, проверку PCB, анализ сетей, историю отчетов и AI-помощник в одном GUI.

## Роль проекта

Проект не заменяет сам EDA-редактор. Его задача — добавить в процесс проектирования более быстрый слой проверки и повторного анализа:

- находить проблемы с позиционными обозначениями, футпринтами, готовностью BOM и DRC уже на этапе схемы
- собирать статистику по компонентам, площадкам, via, трассам, меди и плотным сетям на этапе PCB
- сохранять единые отчеты для нескольких итераций и упрощать сравнение
- использовать собственный endpoint модели для сводок, рекомендаций, чек-листов и произвольных запросов

## Основные возможности

- Единый workbench
  В меню Home, Schematic и PCB доступен только `Open Workbench`, поэтому все действия собраны в одном месте.
- Проверка схемы
  Доступны комплексная проверка, быстрая проверка, schematic DRC и snapshot выделения. Плагин считает компоненты, провода, шины, текст, сетевые маркеры, префиксы designator, отсутствующие designator, отсутствующие footprint и готовность BOM, а затем выдает оценку и рекомендуемые действия.
- Проверка PCB
  Доступны комплексная проверка, быстрая проверка, PCB DRC, snapshot выделения и анализ плотных сетей. Плагин считает компоненты, площадки, via, трассы, дуги, заливки, region и текст, а затем оценивает горячие сети, пробелы в supply-chain данных и риск по via.
- Анализ сетей
  Можно автоматически подсветить самую плотную сеть, выбрать сеть из списка горячих сетей или перейти к сети по имени. Это удобно для проверки питания, земли и критических высокоскоростных линий.
- Система отчетов
  Каждая комплексная проверка, быстрый аудит, запуск DRC и snapshot выделения создают отчет в едином формате. Workbench показывает последний результат и хранит 8 последних записей истории.
- AI Copilot
  Поддерживаются собственный endpoint, имя модели, API Key, дополнительный JSON headers, system prompt и temperature. Встроены действия `сводка последнего отчета`, `рекомендации по исправлению`, `чек-лист проверки` и `произвольный запрос`.

## Примечания по AI

- Плагин не содержит встроенной облачной модели; вызовы идут только на указанный вами endpoint
- Текущий формат запроса совместим с OpenAI `chat/completions`
- В модель передаются последний отчет, текущий контекст документа, горячие сети и активные пороги
- Перед использованием AI нужно включить для расширения разрешение `External Interaction` в JLCEDA

## Сборка и упаковка

```bash
npm install
npm run build
```

Готовые `.eext` файлы появляются в `build/dist/`.
